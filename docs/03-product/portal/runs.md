# Runs

A run is a single execution of a workflow against an environment.

## Run details

Each run captures:

- **Status** — Pass, fail, or error
- **Duration** — Total execution time
- **Steps** — Individual step results with screenshots
- **Video** — Full browser recording
- **Logs** — Detailed execution log

## Viewing runs

Navigate to **Runs** in the Portal sidebar to see your run history.
